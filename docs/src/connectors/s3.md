# AWS S3 Connector

## Register a Connector Instance

```bash
curl -X POST http://localhost:8081/connector-instances \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "id": "s3_data_lake",
    "name": "Data Lake S3",
    "connector_type": "s3",
    "extra_attributes": {
      "bucket": "my-data-lake",
      "region": "us-east-1",
      "access_key_id": "AKIAIOSFODNN7EXAMPLE",
      "secret_access_key": "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"
    }
  }'
```

For EC2 / ECS deployments, omit `access_key_id` and `secret_access_key` to use the instance role credentials automatically.

## Operations

### `get_object`

```json
{
  "connector": "s3_data_lake",
  "operation": "get_object",
  "params": { "key": "data/{{trigger.body.filename}}" }
}
```

Response: `{ "body": "<base64-encoded content>", "content_type": "..." }`

### `put_object`

```json
{
  "connector": "s3_data_lake",
  "operation": "put_object",
  "params": {
    "key": "exports/{{trigger.body.filename}}",
    "body": "{{transform_step.csv_data}}",
    "content_type": "text/csv"
  }
}
```

### `list_objects`

```json
{
  "connector": "s3_data_lake",
  "operation": "list_objects",
  "params": { "prefix": "data/2024/" }
}
```

Response: `{ "objects": [{ "key": "...", "size": N, "last_modified": "..." }] }`

### `delete_object`

```json
{
  "connector": "s3_data_lake",
  "operation": "delete_object",
  "params": { "key": "tmp/{{trigger.body.key}}" }
}
```
